# Telegram Workout Diary Bot (MVP)

Полноценный Telegram-бот для ведения дневника тренировок. Позволяет фиксировать вес, повторения, отказы, сложность подходов и комментарии. Создан для использования во время тренировки.

## Стек технологий
- Node.js 20+
- TypeScript
- Telegraf (Telegram Bot API)
- Prisma ORM + SQLite (с заделом под PostgreSQL)
- pnpm
- Zod для валидации
- Vitest для тестов
- Docker & docker-compose

## Структура проекта
- `src/main.ts` — точка входа
- `src/bot/` — интерфейс бота (Telegraf), сцены, хендлеры, контекст
- `src/modules/` — бизнес-логика (пользователи, тренировки, упражнения, прогресс, экспорт)
- `src/common/` — валидация, утилиты
- `prisma/` — схема БД и seed-скрипт
- `tests/` — unit-тесты
- `data/` — локальные данные (SQLite и сессии)

## Установка и запуск (Local)

1. Установите зависимости:
   ```bash
   pnpm install
   ```

2. Настройте переменные окружения:
   Скопируйте `.env.example` в `.env` и впишите туда токен вашего бота (получить у @BotFather):
   ```
   BOT_TOKEN=ваш_токен
   ```

3. Запустите миграции базы данных и seed-скрипт:
   ```bash
   npx prisma migrate dev --name init
   pnpm seed
   ```

4. Запустите бота в dev-режиме:
   ```bash
   pnpm dev
   ```

## Запуск тестов

Тесты проверяют логику валидации, экспорта и прогресса.
```bash
pnpm test
pnpm typecheck
```

## Запуск в Docker

Для запуска в production/сервере с помощью Docker:
```bash
docker-compose up -d --build
```

База данных и сессии будут сохраняться в volume `data/`.

- [ ] Нажать "Начать тренировку" и выбрать "A (Верх + Ноги)"
- [ ] Записать подход без отказа (вес, повторы, нет, сложность, без комментария)
- [ ] Записать подход с отказом (указать на каком повторении отказ)

- [ ] Отменить текущий ввод (во время записи подхода нажать Отмена)
- [ ] Завершить тренировку (кнопка "Завершить тренировку")
- [ ] Посмотреть историю (кнопка "История" -> "Детали")
- [ ] Посмотреть прогресс по упражнению из тренировки A
